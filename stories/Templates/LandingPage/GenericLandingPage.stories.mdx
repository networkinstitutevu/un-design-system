# Landing Page

A generic landing page has dynamic content list

### Overview

Generic Landing Page is the template where all the content is available for the user. The user can easily search for specific content here. Mock up - https://www.undp.org/news-centre

### Template Structure

The following components are used to create the “Article page” template in the design system
Global header

- Breadcrumbs - Creates navigation links for the current pathname based on an opinionated configuration object.
- Page Hero - Creates boxes that are usually teasing some kind of content like- Image, Text, Subtitle and Breadcrumb.
- Filters and search bar - Allows the user to apply filters to the search section.
- Set of content cards(Results should be filtered based on Filter and search bar) - card of information with some items
- View more button - Call to action that will be performed when the user clicks/touches it
- Standard footer

### Performance

The page performance will be optimized by considering following:

- Lazy load
- Use appropriate media renditions based on the client’s viewpoint.
- Load common/static elements asynchronously (static navigation, menus, countries list, etc)
- Optimize assets loading - inline important CSS, defer bulky assets calls, preconnect to required origins, etc)
- Optimize transitions and animations so above-the-fold elements are always loaded first and available for viewers before the rest of the page is processed.

### Usage

- Copy HTML from the HTML tab of Canvas and include CSS and JS file in ‘CSS and JS References’ section
- Initialize size animation, navigation, language switcher, multi-select, accordion, filter by calling below mentioned functions.
  - expandToSize('.pagehero-full');
  - navigationInitialize();
  - langSwitch();
  - expandSearch();
  - multiSelect();
  - toggleFilter();
  - accordion('[data-accordion="mobile"]', '.footer-panel', 'active');
- Add data-viewport=”true” attribute on elements for the visibility animation
